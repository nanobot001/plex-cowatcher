# Block 00: Project Definition

## Goal

Clarify the project purpose, profile, audience, constraints, stable rules, success criteria, MVP direction, and verification direction.

## Scope

- Fill in `README.md` with a concise project description.
- Fill in `docs/project-charter.md`.
- Record stable project rules in `docs/design/locked-rules.md`.
- Identify the first useful verification command, if known.
- Update `docs/continue-here.md` with current state and next steps.
- Establish the active MVP block sequence starting at `docs/blocks/completed/block-1-1-local-configuration-and-health.md`.

## Out Of Scope

- Building product features.
- Choosing production dependencies unless required to define the project.
- Implementing later blocks.

## Likely Files Or Areas

- `README.md`
- `docs/project-charter.md`
- `docs/design/locked-rules.md`
- `docs/continue-here.md`
- `docs/blocks/completed/block-1-1-local-configuration-and-health.md`

## Acceptance Criteria

- The project purpose, audience, goals, non-goals, MVP direction, constraints, and success criteria are captured.
- The document map is accurate for this project.
- The active MVP block sequence is concrete, bounded, and verifiable.

## Verification

- Review the edited docs for consistency and unresolved TODOs.
