# Block 3-2: Richer Browser UI

## Context
The current local web dashboard provides a minimal view of service health, configured users, and recent prompts. To make Phase 3 fully usable, the UI needs to be expanded to allow deep history searching, pagination, and data exports.

## Acceptance Criteria

- [ ] **Pagination**: Display recent watch events and history with server-side pagination.
- [ ] **Search & Filtering**: Add search bars to filter history by user, media title, library, or date.
- [ ] **CSV Export**: Allow users to click "Export to CSV" to download their watch history for external analysis.
- [ ] **Review Controls**: Allow admins to easily manually dismiss or re-prompt events from the UI.
